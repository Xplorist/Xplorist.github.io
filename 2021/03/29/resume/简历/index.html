  • MacI工程管理系统 | 2017/12 - 2021/03

    • 所属公司：鸿富锦精密电子(成都)有限公司

    • 项目描述：

      1. 系统为MacI制造处建立工程设计文件的记录平台，利于经验积累与传承。系统实现了1.工程文件分功能分模块集中存储、检索和下载；2.工程文件按照专案、阶段和零件三种层次进行分类；3.工程文件线上审核发行，实现无纸化办公；4.严格的用户访问和使用权限控制，保证客户图档等机密资料的安全。
      2. 本系统是基于富士康的iPEG次集团从2010开始开发的老项目进行二次开发而来，2017年末开始一直直到本人离职，该项目长期由本人一人进行项目前后端的二次开发和维护。
      3. 前端技术栈：JSP、jQuery、ExtJS3、DWR、AngularJS、VueJS，后端技术栈：EJB、Struts2、JPA(Hibernate)、Spring、SpringMVC、MyBatis，数据库: Oracle。
      4. 前端的AngularJS和VueJS为本人引入，后端的SSM(SpringMVC+Spring+MyBatis)为本人引入并嵌入到EJB项目中。

    • 责任描述：
      作为项目负责人接手此历史项目，并长期独自对该老项目进行二次开发，三年多内断断续续的需求新增、更迭和维护，对于此项目上自己算是一个全栈开发者，所有MacI产品处后续的新需求从需求分析到前端开发到后端开发再到服务器部署发布项目全部都是由自己一人处理(此项目为EJB项目，单独部署在一台装有Jboss的WindowsSever服务器上，发布项目时涉及到配置文件的修改且操作容易出错，运维嫌部署麻烦，直接把远程连接进行部署的权限给我了)。

  • MacI模治检具招投标系统 | 2019/10 - 2020/04

    • 所属公司：鸿富锦精密电子(成都)有限公司

    • 项目描述：

      1. 基于公司内部的模治检具发单报价的线下模式存在点对点沟通效率低，信息交流不对称，双方价格不统一等缺点，故开发模治检具订单信息服务平台，以实现发单报价流程化、规范化、竞价透明化，从而集中公司优势机器加工资源，提效降本，增加竞争力。
      2. 本项目涉及到公司内部机密文件(产品零件图纸)，故只能在公司内网使用，并无公网在线交易的功能，招投标双方均为公司内部部门，使用集团内部的费用代码进行部门之间的费用交易，非现金交易。
      3. 本项目为前后端分离的项目，前端技术栈：Vue单页应用项目(前端打包后单独部署在Nginx服务器上)，后端技术栈：SpringCloud(Eureka、Zuul、OpenFeign、Config)、SpringBoot、MyBatis，数据库：Oracle。

    • 责任描述：
      作为项目负责人参与整个系统的业务分析、技术选型和功能开发，具体如下：

      1. 与团队成员一起参与招投标业务流程的分析，订单阶段状态的定义，统计模块需要统计的数据类型分析；

      2. 为关联业务系统和论坛系统，仿照CAS(Central Authentication Service)框架为平台开发单点登录中心，并配合用户注册、管理员审核注册用户、管理员管理用户等功能独立成用户模块；

      3. 将项目业务拆分招投标流程模块、用户模块、统计模块、论坛模块，使用SpringCloud搭建微服务系统；

      4. 探索前后端分离部署的方案，最终使用Nginx作为前端单页应用打包后的部署服务器，同时使用Nginx的反向代理去访问后端接口，实现前后端分离部署。

  • MacI门户系统 | 2020/04 - 2021/03

    • 所属公司：鸿富锦精密电子(成都)有限公司

    • 项目描述：

      1. MacI门户系统为MacI制造处各IT系统提供统一入口，并提供对各IT系统的访问用户和访问设备的采集和统计，此系统还集成了其他非IT部门提出软件系统开发需求的用户需求模块，此外，本系统还提供了社区论坛的功能。
      2. 开发环境下，本系统为前后端分离项目；生产环境下，前端项目打包后集成到SpringBoot项目静态资源中，最后SpringBoot项目打包成War包部署到Tomcat中。前端为umi、qiankun、antd、React、TypeScript的微前端项目，业务模块中系统统计和用户需求两个微前端模块为Vue单页应用项目，系统入口和社区论坛模块为React单页应用项目，后端：SpringBoot、Druid, 数据库: Oracle、MySQL。
      3. 本系统难点在于如何在内网环境下收集所有内网IT系统的用户访问数据并统计且不需要每个业务系统提供用户访问数据接口，最后经过本人的探索引入了开源的Matomo分析系统。

    • 责任描述：
      作为项目负责人参与整个系统的设计、开发和技术文档制作，具体如下：

      1. 与团队成员一起参与开发需求分析，功能模块划分，数据库表结构设计；
      2. 探索MacI各业务系统的用户访问数据收集方案，引进了开源的Matomo分析系统；
      3. 对接部门的OpenID-Connect登录中心(当用户登录了门户系统，使用其他业务系统就无需再次登录)，阅读OpenID-Connect的英文官网文档实现了客户端的Web单页应用的隐式认证流程，并为部门开发人员书写OpenID-Connect的中文使用文档；
      4. 使用Druid连接池整合Matomo系统中的MySQL数据库，实现后端双数据源连接(Oracle、MySQL)，查询MySQL数据库来实现统计业务系统的访问数据的后端接口；

个人网站

    描述：用于整理记录自己的技术笔记和日记随笔，技术栈NodeJS+Hexo+MarkDown，采用MarkDown书写文档，开发时基于NodeJS和Hexo的后台服务可以在浏览器中实时查看渲染效果，部署时将MarkDown编译打包成静态HTML发布到Nginx服务器中，配合GitLab的CI/CD，当本地提交MarkDown源码到GitLab服务器，博客服务器中的GitLab Runner组件响应git的提交事件，实现网站自动组建部署发布。

  • GitLab私服

    链接：https://gitlab.xplorist.tech/

    描述: 主要用于同步个人代码、资料和软件项目自动组建部署，由于国内访问GitHub网速较慢(特殊时期甚至需要使用梯子)，于是自己买了阿里云服务器，基于CentOS7搭建了私人GitLab服务器，配置了Nginx的HTTPS协议，使用GitLab的CI/CD功能实现了个人博客网站的自动组建及发布。

  • Matomo私服

    链接：https://www.xplorist.tech/matomo/

    描述：用于个人博客的用户访问数据的收集、统计和分析，使用开源项目Matomo搭建的私服，Matomo是Google Analytics的替代解决方案，优势在于将数据存放在自己的数据库中，开发人员可以使用SQL语句连接MySQL数据库自定义查询并得到需要的数据，基于LNMP技术栈(Linux + Nginx + MySQL + PHP)。
